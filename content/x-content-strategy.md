# X/Twitter Content Strategy

## Content Pillars

### 1. Education
How Clicks works: the 80/20 split, yield routing, why agent USDC sits idle. Target audience: developers who don't know DeFi yield is available to agents.

### 2. Technical
SDK updates, code snippets, MCP server tools, contract architecture. Show real code that runs. Target: builders who want to integrate.

### 3. Ecosystem
x402 payment protocol, Base chain updates, Coinbase Agentic Wallets, Aave V3/Morpho news. Position Clicks within the broader agent economy.

### 4. Community
Integration stories, developer questions answered, open source contributions, GitHub activity. Build trust through participation, not promotion.

### 5. Transparency
Security audit results, contract addresses on Basescan, fee structure, known risks. Differentiate from projects that hide behind vague claims.

## Voice & Tone Rules

### Do
- Technical but accessible. Explain jargon on first use.
- Lead with facts and numbers from our docs/contracts.
- Use real code examples that actually compile.
- Developer humor: dry, subtle, self-aware.
- Acknowledge risks honestly.
- Credit other projects by name.

### Don't
- No "to the moon", no rocket emojis, no "wagmi".
- No fake urgency ("Last chance!", "Don't miss out!").
- No price speculation or token promises (there is no token).
- No superlatives without proof ("best", "revolutionary", "game-changing").
- No emoji spam. One emoji per tweet max, if any.
- No threads that could be one tweet.

## Competitive Positioning

We don't attack competitors. We differentiate by:
1. **Showing code** (no other yield product has an agent SDK)
2. **Transparency** (contract addresses, audit results, known risks)
3. **x402 native** (no competitor integrates with x402)
4. **Non-custodial, no lockup** (verifiable on Basescan)
