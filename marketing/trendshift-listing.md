## TrendShift — Wie es funktioniert

TrendShift (https://trendshift.io) ist eine Alternative zu GitHub Trending, die Repos basierend auf täglichen Engagement-Metriken automatisch erfasst und ranked.

### Automatische Erkennung

TrendShift hat **kein Submit-Formular**. Die Plattform crawlt automatisch GitHub und listet Repos die auf GitHub Trending erscheinen.

### Kriterien für Listing

Ein Repo wird auf TrendShift gelistet wenn es:

1. **Auf GitHub Trending erscheint** (in irgendeiner Sprache oder "All Languages")
2. **Engagement-Metriken erhält:**
   - Stars (täglich/monatlich)
   - Forks
   - Merged Pull Requests
   - Issues & Closed Issues
   - Contributors
   - Commit-Activity

### Badge & Embed

Sobald ein Repo gelistet ist, bietet TrendShift:
- Embed Badge für README.md
- Historische Trending-Daten
- Engagement-Graphen (täglich/monatlich)
- Ranking nach Sprache und Global

## Was für Clicks Protocol nötig ist

Da TrendShift automatisch arbeitet, gibt es keinen "Submit"-Prozess.

### Voraussetzungen:

1. **Repo muss public sein:**
   - github.com/clicks-protocol ist aktuell noch nicht public
   - ⚠️ Sobald das Repo public ist, kann TrendShift es crawlen

2. **Auf GitHub Trending erscheinen:**
   - Das Repo muss organisches Engagement generieren (Stars, Forks, PRs)
   - Sprache-Tags korrekt setzen (z.B. Solidity, TypeScript, Python)
   - Gute README.md mit klarer Beschreibung
   - Topics hinzufügen: `defi`, `ai`, `crypto`, `base`, `yield`, `autonomous-agents`

3. **Engagement-Metriken steigern:**
   - Regelmäßige Commits (zeigt Aktivität)
   - Community-Building (Contributors, Issues, Discussions)
   - Social Proof (externe Links, Mentions)

### Prozess nach Launch:

1. **Repo public machen** auf GitHub
2. **Launch-Kampagne fahren:**
   - Product Hunt
   - Reddit (r/defi, r/cryptocurrency, r/ethereum)
   - Twitter/X (@ClicksProtocol)
   - Discord Community
3. **GitHub Trending anpeilen:**
   - Engagement in den ersten 24-48h ist kritisch
   - Launch-Timing: Dienstag-Donnerstag (best practice)
   - GitHub Actions Badge, Star-Aufruf in README
4. **TrendShift-Tracking:**
   - Sobald auf GitHub Trending → automatisch auf TrendShift
   - Badge von TrendShift in README einbinden (erhöht Social Proof)

- [ ] Repo public machen (GitHub Settings)
- [ ] README.md optimieren (klare Value Prop, Topics, Badges)
- [ ] Launch-Strategie für GitHub Trending vorbereiten
- [ ] Community-Channels aktivieren (Discord, X, Medium)

**Hinweis:** TrendShift listet auch Repos die *noch nicht* auf GitHub Trending waren, aber diese haben dann den Status "This repository has not yet been featured on GitHub Trending". Das Ziel ist also: Auf GitHub Trending kommen → TrendShift Badge nutzen.

## Zusätzliche Insights

### TrendShift Features:
- Repository Engagements Page: zeigt monatliche Metriken
- Developer Profiles: listet Top-Contributors
- Historical Trending Data: wann und wie oft ein Repo auf Trending war
- Badge Embed: Markdown + HTML für externe Websites

### Vergleich GitHub Trending vs. TrendShift:
- **GitHub Trending:** Tägliches Ranking, kein historischer Kontext
- **TrendShift:** Historische Daten, Engagement-Graphen, persistente Rankings

### Marketing-Vorteil:
Einmal auf TrendShift gelistet = dauerhaftes Signal für "dieses Repo war mal relevant". Gut für Social Proof und Discovery auch nach dem Trending-Peak.
