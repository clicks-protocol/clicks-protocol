### Meine Empfehlung zur Funnel-Optimierung

**1. Referral nicht als Front-Door, sondern als Step-2 nutzen.**
Aktuell ist Referral ein Fortgeschrittenen-Narrativ. Für den ersten Nutzer ist wichtiger:
- funktioniert der SDK-Call
- kann ich vertrauen
- sehe ich echten Code und echte Contract-Daten

**2. Erstes öffentliches CTA-Set auf einen einzigen Loop reduzieren.**
Nicht: "yield + referral + teams + bounty".
Sondern:
- Install SDK
- Test quickStart
- Wenn es klappt: invite another agent / operator

**3. Referral-Messaging temporär entschärfen bis Trust-Stack sauber ist.**
Referral extern erst aggressiver spielen, wenn diese 3 Punkte stimmen:
- GitHub public und anschlussfähig
- `.well-known` ohne Placeholder und ohne Falschclaims
- 1 echter public launch surface mit Antworten/Feedback

**4. Minimal-Tracking sofort definieren.**
Ohne neue Infrastruktur reicht vorerst schon ein simples Funnel-Schema:
- source: x / hn / reddit / medium / github / npm
- action_1: docs_view
- action_2: sdk_install
- action_3: quickstart_attempt
- action_4: agent_registered
- action_5: referrer_supplied

**5. Launch die Referral-Story in 2 Ebenen splitten.**
- **Launch-Level:** "idle agent USDC earns yield"
- **Expansion-Level:** "agents can bring other agents on-chain and share fee upside"

Das macht den ersten Pitch klarer und schützt vor MLM-Vibes zu früh.
