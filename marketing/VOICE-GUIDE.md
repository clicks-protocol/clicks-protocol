# Clicks Protocol — Voice & Tone Guide

## Core Identity
**We are:** A technical tool built by developers, for developers. No marketing fluff, no hype. Just working code.

**We sound like:** A senior engineer explaining a clever solution to a peer. Confident but not arrogant. Direct but not rude. Knowledgeable but not condescending.

## Do's ✅

### Language
- **Use technical terms correctly:** "Morpho Blue pools", "Aave V3", "Base L2", "MCP tools", "agent treasury"
- **Be specific:** "80% liquid, 20% earning ~13% APY via Morpho" not "high yields"
- **Explain benefits clearly:** "No lockup means your agent can withdraw anytime" not "flexible liquidity"
- **Use contractions:** "it's", "don't", "we've" — sounds human
- **Short sentences:** One idea per sentence. Periods, not commas.

### Tone
- **Confident but humble:** "We built this because we needed it ourselves"
- **Helpful:** "Here's how to integrate in 3 steps"
- **Transparent:** "2% fee on yield only. No hidden costs."
- **Community-focused:** "Built for the agent economy by the community"

### Formatting
- **Code snippets:** Use backticks for function names, paths, commands
- **Bullet points:** For lists of features, steps, benefits
- **Headers:** Clear section breaks
- **Links:** Always include relevant links (GitHub, docs, website)

## Don'ts ❌

### Language
- **No hype words:** "revolutionary", "game-changing", "pioneering", "disruptive"
- **No corporate buzzwords:** "leverage", "synergy", "ecosystem", "paradigm shift"
- **No emoji spam:** Max 1-2 per post, never in technical explanations
- **No exclamation marks:** "!" feels salesy. Use periods.
- **No Gedankenstriche:** "—" or "–" are LLM tells. Use commas, colons, or periods instead.
- **No rule of three:** "fast, secure, reliable" — pick one strong adjective
- **No vague attributions:** "some say", "many believe" — state facts or don't

### Tone
- **Don't oversell:** "Best yield ever!" → "Currently ~13% APY via Morpho"
- **Don't trash competitors:** "Better than Yearn" → "Different approach: no lockup, lower fees"
- **Don't be defensive:** If criticized, acknowledge and explain
- **Don't use passive voice:** "The yield is earned" → "Your agent earns yield"

### Formatting
- **No bold headers in emails:** Looks like marketing spam
- **No ALL CAPS:** Never shout
- **No excessive formatting:** Keep it clean and readable

## Platform-Specific Guidelines

### X/Twitter
- **Length:** 1-2 sentences max
- **Hashtags:** 1-2 relevant (#DeFi, #AIagents, #Base, #x402)
- **Links:** Always include clicksprotocol.xyz
- **Engagement:** Ask questions, tag relevant accounts
- **Threads:** Each tweet should stand alone, but build narrative

### Reddit
- **Title:** Descriptive, not clickbait
- **Body:** Full explanation, be prepared for technical questions
- **Transparency:** Mention you're the builder
- **Engagement:** Answer all questions honestly

### HN
- **Title:** "Show HN: Clicks Protocol – autonomous yield for AI agents on Base"
- **Body:** Technical details, how it works, why it's interesting
- **No marketing:** Pure tech talk
- **Be ready for criticism:** HN is brutally honest

### Discord
- **Announcements:** Short and sweet
- **Support:** Patient, helpful, technical
- **Community:** Encourage discussion, share wins

### Medium/Substack
- **Depth:** Go deep on one topic
- **Structure:** Clear sections, code examples
- **Audience:** Assume technical readers
- **Call to action:** End with "Try it" or "Give feedback"

## Common Phrases to Use

### Instead of "easy to use"
- "Three lines of code to integrate"
- "npm install and you're ready"
- "MCP tools handle the complexity"

### Instead of "powerful"
- "Handles USDC deposits of any size"
- "Auto-routes to best APY"
- "Real-time yield calculations"

### Instead of "secure"
- "Contracts verified on Basescan"
- "No admin keys, fully on-chain"
- "Audited with Slither (0 critical)"

### Instead of "community-driven"
- "Built with feedback from agent developers"
- "Open source on GitHub"
- "Join the Discord to shape the roadmap"

## Handling Criticism

1. **Acknowledge:** "Good point, thanks for raising this"
2. **Explain:** "Here's why we made that design choice"
3. **Offer solution:** "We're working on improving this"
4. **Invite collaboration:** "Would love your input on GitHub"

## Final Rule
**If it sounds like something a marketing agency would write, don't write it.** If it sounds like something a developer would tell another developer over coffee, write it.
